## READ THIS FIRST — Instructions for Replit Agent

This spec **extends** the existing Healthy Home OS. The `Lead` model, leads API routes, and Leads page already exist.

Your job is to:
1. Add soft delete fields to the Lead schema
2. Update the leads API to support soft delete and audit tracking
3. Make lead rows in the UI clickable (opens a right-side drawer)
4. Add edit and delete flows inside the drawer with confirmation modals

**Build in the order listed. Do not skip phases.**

---

## Safety / Data Protection Rules

Read these before touching anything. These rules override any assumption you make about "cleaning up" existing code.

- Do not delete, drop, recreate, or wipe existing lead data
- Do not drop or recreate the Lead table under any circumstances
- All schema changes must be additive — new fields only, no removals or renames
- Do not remove, rename, or repurpose any existing Lead fields
- Preserve all existing API behavior unless this spec explicitly changes it
- All existing lead list behavior, filters, and working route logic must continue to function after these changes
- Before applying the migration, inspect the current Lead schema and confirm the migration is additive only
- Use an explicit server-side allowlist for editable fields — do not trust arbitrary request body keys
- Only update fields explicitly present in the sanitized request body — never overwrite existing DB values with `undefined`
- Do not append a changeLog entry if no actual field values changed
- Use the same ID type as the existing `User` model for `deletedBy` and `updatedBy` — if user IDs are not plain strings in the current schema, match the existing type

## Build Phase 1 — Schema Changes

**File:** `prisma/schema.prisma`

Add these fields to the **existing** `Lead` model. Do not remove any existing fields.

```prisma
// Add to existing Lead model
isDeleted     Boolean   @default(false)
deletedAt     DateTime?
deletedBy     String?   // userId of who deleted it — match type of User.id
updatedBy     String?   // userId of who last updated it — match type of User.id
changeLog     Json?     // array of change records — see format below
```

**`changeLog` field format** — store as a JSON array, append on every edit:

```json
[
  {
    "changedAt": "2026-01-15T20:00:00Z",
    "changedBy": "user-uuid",
    "changedByName": "Admin",
    "fields": {
      "status": { "from": "new", "to": "quoted" },
      "quoteAmount": { "from": null, "to": 350 }
    }
  }
]
```

After schema changes:
1. Run `npx prisma migrate dev --name lead_soft_delete_and_audit`
2. Run `npx prisma generate`
3. Confirm no errors before continuing.

## Build Phase 2 — Backend Route Changes

**File:** `src/routes/leads.js`

### 2A. Update `GET /api/leads`

Add this filter: **by default, exclude soft-deleted leads.**

```js
// Default behavior — never return deleted leads
where: { isDeleted: false }

// Support optional query param to include deleted (admin only):
// ?includeDeleted=true → remove the isDeleted filter
```

All existing query filters (?status=, ?canvasserId=, ?date=) must continue to work unchanged.

---

### 2B. Update `PUT /api/leads/:id`

**Step 1 — Soft delete gate.** If the lead has `isDeleted = true`, return 404. Do not update a deleted lead.

**Step 2 — Use an explicit allowlist.** Extract only these fields from the request body — ignore everything else silently:

```js
const EDITABLE_FIELDS = [
  'firstName', 'lastName', 'phone', 'email',
  'address', 'city', 'state', 'zip',
  'serviceInterest', 'quoteAmount', 'status',
  'followUpDate', 'notes'
]

const updateData = {}
for (const field of EDITABLE_FIELDS) {
  if (req.body[field] !== undefined) {
    updateData[field] = req.body[field]
  }
}
// updateData now contains only fields explicitly sent — no undefined overwrites
```

**Step 3 — Compute the changeset.** Fetch the current DB record, then compare:

```js
const current = await prisma.lead.findUnique({ where: { id: req.params.id } })
const changedFields = {}
for (const field of Object.keys(updateData)) {
  if (String(updateData[field]) !== String(current[field])) {
    changedFields[field] = { from: current[field] ?? null, to: updateData[field] }
  }
}
```

**Step 4 — If no fields actually changed:** do not write to DB, do not append to changeLog. Return the existing lead as-is with status 200.

**Step 5 — If fields changed:**
1. Build a change entry using `changedFields` in the format shown in Phase 1
2. Append to existing changeLog: `[...(current.changeLog ?? []), newEntry]` — do not overwrite
3. Set `updatedBy = req.user?.id ?? 'system'`
4. Save `updateData` + `updatedBy` + updated `changeLog` in a single Prisma update call
5. Return the full updated lead

---

### 2C. Update `DELETE /api/leads/:id`

**Soft delete gate.** If the lead already has `isDeleted = true`, return success (idempotent — do not return 404, do not hard delete).

**Replace hard delete with soft delete.** Do not use `prisma.lead.delete()`.

```js
await prisma.lead.update({
  where: { id: req.params.id },
  data: {
    isDeleted: true,
    deletedAt: new Date(),
    deletedBy: req.user?.id ?? 'system'
  }
})
```

Return: `{ success: true, message: "Lead deleted." }`

This lead will no longer appear in `GET /api/leads` (filtered by `isDeleted: false`).

## Build Phase 3 — Frontend: Clickable Lead Rows

**File:** `client/src/pages/Leads.jsx`

Make the following changes to the existing Leads page. Do not rebuild the whole page. Do not remove existing filters, sort behavior, or list rendering logic.

---

### 3A. Make each lead row clickable

Add an `onClick` handler to each lead row:

```jsx
<tr onClick={() => handleLeadClick(lead)} style={{ cursor: 'pointer' }}>
```

Add state to the Leads page:
```js
const [selectedLead, setSelectedLead] = useState(null)
const [drawerOpen, setDrawerOpen] = useState(false)
const [drawerLoading, setDrawerLoading] = useState(false)
const [drawerError, setDrawerError] = useState(null)
```

**`handleLeadClick` function:**
```js
async function handleLeadClick(lead) {
  setSelectedLead(lead)   // show drawer immediately with data we already have
  setDrawerOpen(true)
  setDrawerLoading(true)
  setDrawerError(null)
  try {
    const res = await fetch(`/api/leads/${lead.id}`)
    if (!res.ok) throw new Error('Failed to load lead')
    const full = await res.json()
    setSelectedLead(full)  // replace with full detail once loaded
  } catch (err) {
    setDrawerError('Could not load lead detail. Please try again.')
  } finally {
    setDrawerLoading(false)
  }
}
```

**After a successful edit, update both the drawer and the list:**
```js
function handleLeadUpdated(updatedLead) {
  setSelectedLead(updatedLead)                                // update drawer
  setLeads(prev => prev.map(l =>                             // update list row
    l.id === updatedLead.id ? { ...l, ...updatedLead } : l
  ))
}
```

**After a successful delete, update the list and close drawer:**
```js
function handleLeadDeleted(deletedId) {
  setLeads(prev => prev.filter(l => l.id !== deletedId))    // remove from list
  setDrawerOpen(false)
  setSelectedLead(null)
}
```

Pass `handleLeadUpdated` and `handleLeadDeleted` as props into `LeadDrawer`.

## Build Phase 4 — Edit Flow

**File:** `client/src/components/LeadDrawer.jsx`

---

### 4A. Edit mode

Add state:
```js
const [editMode, setEditMode] = useState(false)
const [formData, setFormData] = useState({})
```

When **Edit** button is clicked:
- Set `editMode = true`
- Initialize `formData` with a copy of current lead values for editable fields only
- All editable fields switch from display text to input fields (in-place)
- Action bar changes to show **Save** and **Cancel** buttons

**Editable fields:**
- firstName, lastName
- phone, email
- address, city, state, zip
- serviceInterest (dropdown: house_wash / driveway / bundle)
- quoteAmount (number input)
- status (dropdown: new / quoted / follow_up / sold / lost)
- followUpDate (date input)
- notes (textarea)

**Non-editable fields (display only, never become inputs):**
- id, createdAt, source, canvasserId, changeLog, updatedBy

**Cancel button:** resets `formData`, sets `editMode = false`. No modal needed.

---

### 4B. Save confirmation modal

When **Save** is clicked in edit mode:

1. Compute changed fields by comparing `formData` to original `lead` prop values
2. If nothing changed, show toast: `"No changes to save."` — do not open modal
3. If changes exist, open the **Edit Confirmation Modal**

**Edit Confirmation Modal content:**
```
Title: "Save Changes?"

Body: Show a table of changed fields only:
  Field        | Before      | After
  -------------|-------------|-------
  Status       | new         | quoted
  Quote Amount | —           | $350

Buttons:
  [Cancel]   [Save Changes]
```

When **Save Changes** is confirmed:
- PUT to `/api/leads/:id` with only the changed fields in the body (not the full form)
- On success: call `onLeadUpdated(responseData)`, exit edit mode, show toast: `"Lead updated."`
- On error: show toast: `"Update failed. Please try again."` — stay in edit mode
